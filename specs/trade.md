# 단일종목 레버리지/인버스 매매 로직 명세서

이 문서는 SK하이닉스 등 **단일종목 2배 레버리지/인버스 상품**을 5분봉 기준으로 매매하기 위한
규칙 기반(rule-based) 로직 명세서입니다. 코드가 아니라 로직 설계 문서이며, 이 문서를 참고해서
백테스트/실시간 시그널 스크립트를 구현할 때 기준으로 사용합니다.

---

## 0. 전제 및 데이터

- 타임프레임: 5분봉 (다른 타임프레임 적용 시 파라미터 재검증 필요)
- 대상: 단일종목 레버리지 상품(롱 익스포저), 인버스 상품(숏 익스포저) — 현물 공매도 대신 인버스 상품으로 숏 노출을 대신함
- 필요 지표
  - `DEMA(60)`: 추세 방향 판단용 (후행지표 — 전환 "확인"용으로 사용, 전환 "예측"용 아님)
  - `CCI(20)` + `Signal(10)`: 메인 타이밍 지표
  - `Stochastic %D(3)`: 과열/과매도 보조 필터
  - `Volume`: 저점/전환 신뢰도 보강용
  - (선택) 일목균형표 선행스팬1/2: 지지/저항 참고용

## 1. 파라미터 기준값

```yaml
dema:
  period: 60
  role: "confirmation_only"   # 진입/청산의 단독 트리거로 쓰지 않음. 기울기 전환 확인용.

cci:
  period: 20
  signal_period: 10
  zero_line: 0
  overbought_warning: 150      # 1차 경고(부분 익절) 트리거
  overbought_extreme: 200      # 참고용 상한
  oversold_warning: -150       # 인버스 1차 경고(부분 익절) 트리거
  oversold_extreme: -200       # 참고용 하한
  normal_trend_zone: [100, 150]   # 상승 추세 진행 중, 홀딩 구간
  normal_trend_zone_inverse: [-150, -100]  # 하락 추세 진행 중, 홀딩 구간

stochastic:
  d_period: 3
  overbought: 80
  oversold: 20
  role: "entry_filter_and_secondary_confirmation"
  # 진입 시: 80 이상이면 레버리지 신규 진입 보류 (추격매수 방지)
  #          20 이하면 인버스 신규 진입 보류 (추격 숏 방지)
  # 1차 경고 보조 확인용으로는 완화 기준(60/40)도 참고 가능하나 단독 트리거 금지

position_sizing:
  max_allocation_pct: 10   # 계좌 대비 최대 비중 (2배 상품이므로 실질 노출은 2배)
```

## 2. 레버리지(롱) 진입 조건 — 3-of-3 규칙

**아래 3개 조건이 모두 동시에 충족될 때만 신규 진입.** 2개만 충족 시 관망(워치리스트에만 등록).

| # | 조건 | 판정 로직 |
|---|---|---|
| 1 | DEMA 기울기 우상향 전환 | `DEMA[t] - DEMA[t-1] > DEMA[t-1] - DEMA[t-2]` (기울기가 이전 대비 커짐, 즉 우상향 강화) |
| 2 | CCI 0선 돌파 또는 Signal 골든크로스 | `CCI[t-1] <= 0 and CCI[t] > 0` **또는** `CCI[t-1] <= Signal[t-1] and CCI[t] > Signal[t]` (0선 위에서) |
| 3 | 가격 DEMA 상향 돌파 + 직전 고점 갱신 | `Close[t] > DEMA[t] and Close[t] > max(High[t-N:t-1])` (N=최근 스윙 고점 탐색 구간, 기본 10봉) |

**진입 필터 (거부 조건)**
- `Stochastic_D[t] >= 80` 이면 신규 진입 보류 (이미 과열)

---

## 3. 인버스(숏 대체) 진입 조건 — 3-of-3 규칙 (대칭)

| # | 조건 | 판정 로직 |
|---|---|---|
| 1 | DEMA 기울기 하향 전환 | `DEMA[t] - DEMA[t-1] < DEMA[t-1] - DEMA[t-2]` (기울기가 이전 대비 더 가파르게 하락) |
| 2 | CCI 0선 이탈 또는 Signal 데드크로스 | `CCI[t-1] >= 0 and CCI[t] < 0` **또는** `CCI[t-1] >= Signal[t-1] and CCI[t] < Signal[t]` (0선 아래에서) |
| 3 | 가격 DEMA 하향 돌파 + 직전 저점 갱신(신저가) | `Close[t] < DEMA[t] and Close[t] < min(Low[t-N:t-1])` |

**진입 필터 (거부 조건)**
- `Stochastic_D[t] <= 20` 이면 신규 진입 보류 (이미 과매도 — 뒤늦은 추격 숏 방지)
- **거래량 확인 권장**: 하락 전환 시 거래량이 직전 평균 대비 확연히 증가했는지 확인 (진짜 매물 출회 vs 단순 눌림 구분). 거래량 증가 없이 조건만 충족되면 신뢰도 낮음 → 포지션 축소 고려.

---

## 4. 청산 로직 — 2단계(경고 → 확정)

### 4-1. 레버리지 보유 중 청산

| 단계 | 트리거 | 액션 |
|---|---|---|
| 1차 경고 (부분 익절) | `CCI[t] >= 150` 이고 나서 `Signal` 데드크로스 발생 (단, DEMA 기울기·고점 갱신 여부는 아직 살아있어도 됨) | 보유 물량 1/3~1/2 익절, 잔여 홀딩 |
| 2차 확정 (전량 청산 → 인버스 전환) | 아래 중 **2개 이상** 동시 충족:<br>① `CCI[t] < 0` (0선 이탈)<br>② DEMA 기울기 하향 전환 (3장 조건1과 동일 로직)<br>③ 가격이 DEMA 하향 돌파 + 직전 저점 이탈 | 잔여 물량 전량 매도. 3장(인버스 진입) 조건 재확인 후 인버스 신규 진입 |

**주의**: `CCI`가 단순히 고점 대비 살짝 꺾이는 것(예: +100대 초반에서의 되돌림)은 **눌림목**으로 간주하고 청산하지 않음. 1차 경고 기준(`CCI >= 150`)에 못 미치면 그대로 홀딩.

### 4-2. 인버스 보유 중 청산 (대칭 로직)

| 단계 | 트리거 | 액션 |
|---|---|---|
| 1차 경고 (부분 익절) | `CCI[t] <= -150` 이고 나서 `Signal` 골든크로스 발생 (DEMA 기울기는 아직 하향 유지 가능) | 보유 물량 1/3~1/2 익절, 잔여 홀딩 |
| 2차 확정 (전량 청산) | 아래 중 **2개 이상** 동시 충족:<br>① `CCI[t] > 0` (0선 상향 돌파)<br>② DEMA 기울기 우상향 전환<br>③ 가격이 DEMA 상향 돌파 + 직전 반등고점 갱신 | 잔여 물량 전량 매도. 2장(레버리지 진입) 조건 재확인 후 재진입 판단 |

**주의**: `Stochastic %D`가 20 이하에서 한 번 반등한 것만으로는 청산하지 않음 (아래 5장 "저점 판단" 기준 참고 — 최소 2개 이상의 저점 신호가 겹쳐야 신뢰).

---

## 5. 홀딩 유지 조건 (포지션 보유 중 매 봉마다 재확인)

### 레버리지 홀딩 유지 (3개 다 살아있어야 홀딩)
1. DEMA 기울기 우상향 유지
2. CCI가 0선 위 유지 (일시 되돌림 허용, 0선 이탈 시에만 경고)
3. 가격이 DEMA 위 유지, 저점 갱신 없이 유지

### 인버스 홀딩 유지 (3개 다 살아있어야 홀딩)
1. DEMA 기울기 하향 유지
2. CCI가 0선 아래 유지 (일시 반등 허용, 0선 돌파 시에만 경고)
3. 가격이 DEMA 아래 유지, 고점 갱신 없이 유지

## 6. 저점(바닥) 판단 — 인버스 청산/레버리지 재진입 신뢰도 보강용

단일 신호로 판단하지 않고, 아래 중 **최소 2개 이상 동시 충족** 시에만 "신뢰도 높은 저점"으로 인정.

| # | 항목 | 판정 로직 |
|---|---|---|
| 1 | 스윙 로우 구조 전환 (Higher Low) | 직전 저점을 하회하다가, 다음 저점이 이전 저점보다 높게 형성 (`Low[t] > Low[prev_swing_low]` 이면서 `Low[prev_swing_low] < Low[prev_prev_swing_low]`) |
| 2 | 강세 다이버전스 | `Price`는 신저가 갱신(`Low[t] < Low[t-k]`) 하는데 `CCI[t] > CCI[t-k]` 또는 `Stochastic_D[t] > Stochastic_D[t-k]` (지표는 저점 높아짐) |
| 3 | Stochastic %D 반등 | `Stochastic_D[t] <= 20` 구간에서 골든크로스 발생 |
| 4 | 거래량 동반 | 하락 막바지 구간에서 거래량이 직전 N봉 평균 대비 급증 + 아래꼬리 긴 캔들 (투매 후 매수세 유입 패턴) |

- **약한 신호** (③만 충족): 청산/재진입 보류, 관망
- **강한 신호** (①+② 또는 ①②③ 조합): 4장의 "2차 확정 청산" 또는 2장의 "레버리지 재진입" 검토

---

## 7. 리스크 관리 규칙

- 신규 포지션: 계좌 대비 5~10% 이내 (2배 상품 특성상 실질 익스포저는 그 2배)
- 물타기(추가매수) 금지: 특히 인버스 손실 중 추가 매수 금지. 홀딩 조건(5장)이 유지되는 동안만 기존 포지션 유지, 조건 붕괴 전 추가 진입 안 함
- 보유 기간: 원칙적으로 당일 청산 지향 (음의 복리효과로 인한 다음날 재정산 리스크 회피)
- 다중 조건 요구 이유: 단일 지표(CCI 또는 Stochastic 단독)만으로 매매 시 휩쏘(whipsaw) 빈발 → 반드시 DEMA(추세) + CCI(모멘텀) + 필요시 거래량/구조(저점 판단)를 조합

## 8. 요약 다이어그램 (상태 전이)

```
[관망] --(2장 3-of-3 충족)--> [레버리지 보유]
[레버리지 보유] --(CCI>=150 & Signal 데드크로스)--> [레버리지 1차 부분익절]
[레버리지 1차 부분익절] --(2개 이상: CCI<0 / DEMA꺾임 / 가격DEMA하향돌파+저점이탈)--> [레버리지 전량청산]
[레버리지 전량청산] --(3장 3-of-3 재확인)--> [인버스 보유]
[인버스 보유] --(CCI<=-150 & Signal 골든크로스)--> [인버스 1차 부분익절]
[인버스 1차 부분익절] --(2개 이상: CCI>0 / DEMA기울기전환 / 가격DEMA상향돌파+고점갱신, 6장 저점판단으로 신뢰도 보강)--> [인버스 전량청산]
[인버스 전량청산] --(2장 3-of-3 재확인)--> [레버리지 보유] (재진입 시작)
```

## 9. 구현 시 참고사항 (Claude Code 용)

- 위 로직을 함수화할 때 각 조건은 독립된 boolean 반환 함수로 분리 (`is_dema_slope_up()`, `is_cci_cross_up()`, `is_price_break_dema_up()` 등) 후 조합 조건에서 AND/OR 결합
- 스윙 고점/저점 탐색(N봉)은 파라미터화하여 백테스트로 최적값 탐색 권장 (기본값 10봉으로 시작)
- 이 로직은 SK하이닉스 5분봉을 기준으로 정성적으로 도출된 규칙이며, 실제 구현 후 과거 데이터로 백테스트하여 임계값(150/-150, 80/20 등)을 재검증할 것
